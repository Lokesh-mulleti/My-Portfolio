# Lokesh Mulleti Portfolio Project Documentation

## 1. Project Overview

This project is a responsive personal portfolio website for Lokesh Mulleti. It was built using React.js and Vite, with custom CSS for layout, colors, responsiveness, animations, dark mode, cards, and visual styling.

The portfolio presents:

- Personal introduction
- Contact links
- Resume download
- Technical skills
- Internship experience
- Featured projects
- Education
- Certifications
- Achievements
- Contact call-to-action

The project is designed to work across desktop, tablet, and mobile screens.

## 4. File-by-File Explanation

### `package.json`

This file defines the project name, scripts, dependencies, and module type.

Important fields:

- `name`: Project name, currently `lokesh-portfolio`.
- `private`: Prevents accidental publishing to npm.
- `version`: Project version.
- `type: "module"`: Allows modern ES module imports.
- `scripts`: Commands used to run and build the project.
- `dependencies`: External packages needed by the app.

Scripts:

```json
"dev": "vite --host 127.0.0.1"
```

Starts the development server locally.

```json
"build": "vite build"
```

Creates the production-ready files inside `dist/`.

```json
"preview": "vite preview --host 127.0.0.1"
```

Previews the production build locally.

Dependencies:

- `react`: Core React library.
- `react-dom`: Renders React into the browser DOM.
- `vite`: Fast build tool and dev server.
- `@vitejs/plugin-react`: React support for Vite.
- `lucide-react`: Icon components.

### `package-lock.json`

This file locks the exact versions of installed packages.

Why it matters:

- Ensures the same dependency versions are installed again.
- Helps avoid version mismatch problems.
- Is automatically created by `npm install`.

You normally do not edit this file manually.

#### `useTheme()` Hook

```js
function useTheme() {
  const [theme, setTheme] = React.useState(() => localStorage.getItem('theme') || 'light');
  ...
}
```

This custom hook manages light and dark mode.

Key ideas:

- `useState`: Stores the current theme.
- `localStorage`: Saves the selected theme in the browser.
- `useEffect`: Applies the theme to the HTML element.

This line sets the theme:

```js
document.documentElement.dataset.theme = theme;
```

The CSS reads this using:

```css
:root[data-theme='dark']
```

## 5. Important Keywords and Their Meaning

### React

A JavaScript library used to build user interfaces using components.

### Component

A reusable piece of UI.

Examples in this project:

- `Header`
- `HeroVisual`
- `SectionHeader`
- `App`

### JSX

JavaScript syntax that looks like HTML.

Example:

```jsx
<h1>Lokesh Mulleti</h1>
```

### Props

Data passed into a component.

Example:

```jsx
<SectionHeader eyebrow="Projects" title="Selected builds">
```

Here, `eyebrow` and `title` are props.

### State

Data that can change while the app runs.

Example:

```js
const [open, setOpen] = React.useState(false);
```

This controls the mobile menu.

### Hook

A React function that adds behavior to components.

Hooks used:

- `useState`
- `useEffect`

### `useState`

Stores changing data.

Used for:

- Theme value
- Mobile menu open/closed state

### `useEffect`

Runs code after React renders.

Used for:

- Applying the selected theme to the HTML element
- Saving the theme to local storage

### `localStorage`

Browser storage used to remember the selected theme.

### `map()`

JavaScript array method used to render repeated UI.

Examples:

- Navigation items
- Contact links
- Skill cards
- Project cards
- Certifications
- Achievements

### CSS Variables

Reusable CSS values.

Example:

```css
--accent: #12b886;
```

Used like:

```css
color: var(--accent);
```

### Flexbox

CSS layout system used for one-dimensional layouts.

Used for:

- Header
- Buttons
- Tags
- Contact links
- Hero metadata

### CSS Grid

CSS layout system used for two-dimensional layouts.

Used for:

- Hero section
- Skill grid
- Project grid
- Education split layout
- Hero visual panel

### Media Query

CSS rule that applies styles at specific screen widths.

Used to make the site responsive.

### Responsive Design

Design approach that makes the website work on desktop, tablet, and mobile screens.

### Vite

Fast frontend development and build tool.

Used to:

- Run local dev server
- Bundle React app for production
- Serve public assets

### Build

The process of converting source files into production-ready files.

Command:

```bash
npm.cmd run build
```

Output folder:

```text
dist/
```

## 9. How the Website Works from Start to Finish

1. Browser opens `index.html`.
2. `index.html` loads `/src/main.jsx`.
3. `main.jsx` imports React, icons, and `styles.css`.
4. React renders the `App` component into `<div id="root"></div>`.
5. `App` renders all portfolio sections.
6. Data arrays such as `projects`, `skillGroups`, and `contactLinks` generate repeated cards and links.
7. `styles.css` applies the visual design.
8. Media queries adjust layouts for tablet and mobile.
9. Theme toggle changes `data-theme` on the HTML element.
10. CSS variables switch the page between light and dark mode.
